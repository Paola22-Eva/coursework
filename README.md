# Курсовая работа: Объяснимость модели (Grad-CAM / LIME)

## Описание проекта
Данный проект реализует обучение сверточной нейронной сети (CNN) для классификации рукописных цифр MNIST и визуализацию принятия решений модели с помощью методов Grad-CAM и LIME.  
Цель работы — интерпретировать, какие области изображения влияют на решение модели, сравнить два метода объяснимости и проанализировать типичные ошибки.

## Описание ключевых файлов

### config/config.py
Содержит класс `Config` с настройками:
- путь к данным, размер батча, количество эпох, устройство (cuda/cpu)
- seed для воспроизводимости
- параметры визуализации

### data/data_loader.py
Загружает датасет MNIST, применяет нормализацию и разбиение на train/test.  
Функция `get_sample_images()` возвращает фиксированное количество тестовых изображений для анализа.

### models/model.py
Определяет класс `CNNModel` — сверточную нейросеть с тремя сверточными блоками.  
В модели добавлены BatchNorm и Dropout для регуляризации.  
Последний сверточный блок выделен в отдельный `conv5_block` для корректной работы Grad-CAM.

### training/trainer.py
Класс `Trainer` реализует:
- обучение с вычислением loss на каждой эпохе
- валидацию на тестовой выборке
- сохранение лучшей модели
- логирование потерь и точности

### evaluation/metrics.py
Содержит функцию `calculate_metrics()`, возвращающую accuracy, weighted F1-score и матрицу ошибок.

### evaluation/error_analysis.py
Функции `find_correct_and_errors()` и `analyze_errors()`:
- находят правильные и ошибочные предсказания модели на тестовой выборке
- возвращают изображения и метки для дальнейшего анализа
- выводят статистику по ошибкам

### explainability/gradcam.py
Класс `GradCAM` реализует метод Grad-CAM:
- регистрация хуков для захвата активаций и градиентов целевого слоя
- генерация тепловой карты для заданного изображения и целевого класса
- в реализации используется слой `conv5_block` (после ReLU) и ReLU не применяется к итоговой карте для лучшей интерпретации

### explainability/lime_explainer.py
Класс `LIMEExplainer` оборачивает библиотеку lime:
- `predict_proba()` адаптирует предсказания модели для LIME
- `explain()` генерирует объяснение для изображения
- `get_heatmap()` извлекает тепловую карту важности

### visualization/plots.py
Функции для построения:
- кривых обучения `plot_training_curves()`
- матрицы ошибок `plot_confusion_matrix()`
- t-SNE визуализации эмбеддингов `plot_tsne()`
- сетки объяснений `plot_explanations_grid()`
- анализа ошибок `plot_error_analysis()`

### visualization/utils_viz.py
Вспомогательные функции для работы с визуализацией:

- denormalize() – приводит нормализованные тензоры обратно к диапазону [0,1] для отображения
- overlay_heatmap() – накладывает тепловую карту Grad-CAM на исходное изображение
- overlay_lime() – накладывает цветную маску LIME (зелёные сегменты) на изображение

### main.py
Главный скрипт, запускающий полный пайплайн:
1. Инициализация конфигурации и seed
2. Загрузка данных
3. Создание модели
4. Обучение (или загрузка чекпоинта)
5. Оценка качества
6. Анализ ошибок и формирование наборов правильных/ошибочных изображений
7. Визуализация t-SNE
8. Генерация Grad-CAM и LIME для правильных и ошибочных примеров
9. Сохранение итоговых визуализаций

## Установка и запуск

### Требования
- Python 3.8+
- PyTorch 1.12+
- CUDA (опционально, для ускорения на GPU)
- Другие зависимости указаны в `requirements.txt`

### Установка зависимостей
```bash
pip install -r requirements.txt
```

### Запуск
```bash
python main.py
```

Программа автоматически:
- загрузит датасет MNIST (если он не скачан ранее)
- создаст папки `checkpoints/` и `results/figures/`
- обучит модель (или загрузит уже сохраненную)
- сгенерирует все визуализации в папку `results/figures/`

### Ожидаемые результаты
После выполнения в папке `results/figures/` появятся:
- `training_curves.png` – графики обучения
- `confusion_matrix.png` – матрица ошибок
- `error_analysis.png` – примеры ошибочных предсказаний
- `tsne.png` – визуализация эмбеддингов в 2D
- `correct_explanations.png` – объяснения Grad-CAM и LIME для правильных примеров (до 5)
- `error_explanations.png` – объяснения для ошибочных примеров (до 5)

## Примечания
- В реализации Grad-CAM используется слой `conv5_block`, а ReLU не применяется к тепловой карте. Это позволяет получить более информативные карты.
- Если на вашей машине нет GPU, код автоматически использует CPU.
